### Code Style

- Use TypeScript for all new code
- Follow the existing code formatting (Prettier is configured)
- Use descriptive variable and function names
- Keep components small and focused

### Component Guidelines

- Follow the existing component structure in `/client/src/components`
- Use Radix UI primitives when building new components
- Implement proper error boundaries
- Ensure responsive design

### Styling

- Use Tailwind CSS for styling
- Follow the existing color scheme and design tokens
- Keep animations subtle and smooth
- Test in both light and dark themes (when applicable)

- Run `pnpm check` to ensure TypeScript compliance
- Test your changes in different screen sizes

## ğŸ“ Submitting Changes

### Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Update the README if you've added features
4. Submit a clear pull request with:
   - Descriptive title
   - Summary of changes
   - Screenshots if applicable
   - Any breaking changes

### Commit Messages

Use clear, descriptive commit messages:
- `feat: add new feature`
- `fix: resolve issue description`
- `docs: update documentation`
- `style: improve formatting`
- `refactor: improve code structure`
